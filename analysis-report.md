## 1. 분석 개요

### 분석 대상
- **코드베이스**: 99개 소스 파일, 28개 라우트 (Next.js 15 + TypeScript + Tailwind v4)
- **클라이언트 피드백**: 음성 녹취록 (15분 18초) — 온보딩, 상담 흐름, UI/UX, 신규 기능 전반

### 분석팀 구성
| 분석가 | 담당 영역 | 주요 발견 |
|--------|----------|----------|
| domain-analyst | 용어, 비즈니스 로직, 워크플로우 | 용어 오류 10건, 온보딩 순서 오류 |
| ux-analyst | 전 페이지 UX/UI 심층 분석 | 12개 페이지 분석, 우선순위 상 7건 |

### 핵심 결론

**즉시 수정 필요 (P0)**: 용어 변경, 온보딩 순서, 컬러 참고 제거, 서비스 목록 순서
**대규모 개선 필요 (P1)**: 아이콘/이미지 직관성 강화, 캔버스 손 일러스트 교체, 기록 탭 레이아웃 재설계, 최종 확인 페이지 재설계, 테마 개편, 파츠 시스템 개편

## 2. 도메인/용어 분석

### 2.1 용어 매핑표

| 현재 (잘못됨) | 올바른 업계 용어 | 위치 | 비고 |
|--------------|-----------------|------|------|
| `단색톤` (solid_tone) | **원컬러** | service-options.ts, consultation.ts, mock-dashboard.ts | 클라이언트 직접 요청 |
| `풋` (foot) | **페디큐어** | BODY_PART_OPTIONS | '풋'은 업계 미사용 |
| `패디큐어` | **페디큐어** | service-options.ts | 오타 (일부 표기 불일치) |
| `디자인 범위` | **아트 범위** 또는 **시술 범위** | DesignScopeSelector | '디자인 범위'는 모호 |
| `자샵 오프` / `타샵 오프` | **자샵오프** / **타샵오프** | OFF_TYPE_OPTIONS | 업계에서 붙여 씀 |
| `S/A/B 등급 파츠` | **사장님 직접 등록** | PARTS_GRADE_OPTIONS | 등급 비직관적 |
| `솔리드` (expression) | **기본** 또는 **단색** | EXPRESSION_OPTIONS | 일반 사장님에겐 낯선 표현 |

### 2.2 온보딩 순서 오류

**현재** (논리적 모순):
```
매장정보 → 가격 설정 → 서비스 선택 → 추가금 → 시간 → 완료
```
- 서비스를 뭘 하는지도 모른 채 가격을 먼저 설정하는 비논리적 흐름

**수정안** (클라이언트 확정):
```
매장정보 → 서비스 선택 → 기본 가격 → 추가금 → 시간 → 테마 → 완료
```

### 2.3 서비스 목록 순서

**현재**: 젤네일, 패디큐어, 속눈썹연장, 왁싱, 아크릴, 네일아트, 케어, 제거

**수정**: 젤네일 → 페디큐어 → 네일아트 → 케어 → 제거 → 속눈썹연장 → 왁싱 → 아크릴

핵심 서비스(네일 계열)를 앞에, 부가 서비스를 뒤에 배치. 기본 서비스(젤네일/페디큐어)는 비활성화 불가 처리 필요.

### 2.4 파츠/컬러 시스템 문제

**S/A/B 등급 시스템의 문제점**:
1. 사장님이 "S등급"이 뭔지 즉시 이해 어려움
2. 가게마다 파츠 종류/가격이 다른데 고정 등급은 확장성 없음
3. 클라이언트 명시적 피드백: "사장님이 직접 등록하는 방식"으로 변경

**사장님 직접 등록 방식 제안**:
```typescript
interface CustomPart {
  id: string;
  name: string;         // "스와로브스키 큐빅", "진주 파츠" 등
  pricePerUnit: number; // 개당 가격 직접 입력
  isActive: boolean;
}
```

### 2.5 대시보드 KPI 문제

현재 KPI(총 상담 127건, 총 매출, 평균 금액, 고객 수)는 **숫자 나열일 뿐 실행 가능한 인사이트가 아님**.

**사장님에게 필요한 KPI**:
- 오늘 시술/예약 건수, 오늘 매출
- 이번 달 매출 목표 진행률
- 재방문율, 인기 시술 TOP 3
- 아직 안 온 단골 목록 (30일+ 미방문)
- 할인 적용 비율, 예약금 수금률

## 3. UX/UI 분석

### 3.1 공통 문제점

| 문제 | 심각도 | 영향 범위 |
|------|--------|----------|
| **전반적 아이콘 부재** | 🔴 높음 | 전체 페이지 — 외국인 직관성 핵심 |
| **가로 여백 과다** | 🔴 높음 | 전체 — max-w 미설정으로 넓은 화면 중앙 빈 공간 |
| **스크롤 문제** | 🟡 중간 | 전체 — 일부 페이지 하단 콘텐츠가 탭바에 가려짐 |
| **이모지 아이콘 사용** | 🟡 중간 | step2 등 — OS별 렌더링 차이, 일관성 부족 |

**해결 방향**:
- 전체 컨테이너에 `max-w-md mx-auto` 적용
- SelectCard 레이아웃: 가로(아이콘좌+텍스트우) → 세로(아이콘상+텍스트하)
- 이모지 → SVG 인라인 아이콘 교체

### 3.2 페이지별 핵심 이슈

#### 홈 (`/home`) — 우선순위: 상
- 오늘의 요약 카드에 아이콘 완전 부재
- 통계 카드 숫자만 크게 표시, 시각적 계층 없음
- **개선**: 각 셀에 SVG 아이콘 추가 (위) + 숫자 (중) + 라벨 (아래) 패턴

#### 기록 탭 (`/records`) — 우선순위: 상 (클라이언트 직접 지적)
- **핵심 문제**: `justify-between` 레이아웃으로 좌/우 두 블록만 존재 → 카드 중앙 공간 항상 비어있음
- 클라이언트 원문: "텍스트가 좌우 치우쳐져 중앙이 비어있는 잘못된 UI"
- **개선**: 중앙 정렬 카드 레이아웃 (`max-w-lg mx-auto`), 카드 내부 정보 재배치

#### 상담 기본조건 (`/consultation/step1`) — 우선순위: 상
- 네일 쉐입 SVG 아이콘이 비현실적 (단순 rect/path 조합)
- OffSelector/ExtensionSelector에 아이콘 없이 텍스트만
- BodyPart 손 SVG가 단순한 선 그래프 수준
- **개선**: 쉐입 SVG를 사실적으로 교체, 각 셀렉터에 직관적 아이콘 추가

#### 캔버스 (`/consultation/canvas`) — 우선순위: 상 (클라이언트 직접 지적)
- HandIllustration SVG가 비현실적 — 직선적 palm, 균일한 손가락, 스킨 컬러 없음
- 클라이언트 원문: "진짜 손 이미지를 넣어줘야 돼"
- **개선**: 실제 손 이미지 또는 고품질 일러스트로 교체, 피부톤 적용

#### 최종 확인 (`/consultation/summary`) — 우선순위: 상 (클라이언트 직접 지적)
- 최종 금액이 얇은 배너에 text-xl로 작게 표시
- 정보 순서: 담당선생님 → 고객정보 → ... → 가격 (가격이 맨 아래)
- **개선**:
  - 가격/시간을 히어로 카드로 (text-4xl, 그라데이션 배경)
  - 정보 순서: 가격 → 시간 → 디자인 → 시술조건 → 추가옵션 → 고객정보
  - 가격 상세는 Accordion(접었다 펼치기) 방식

#### 고객 상세 (`/customers/[id]`) — 우선순위: 중
- 시술 성향 태그가 너무 길음 (미등록 카테고리도 나열 → 스크롤 과다)
- **개선**: 등록된 태그만 표시, 1-2줄로 제한, 미등록 카테고리 숨김

#### 대시보드 (`/dashboard`) — 우선순위: 중
- KPI 카드 정적 표시만, 클릭/드릴다운 없음
- 실행 가능한 인사이트 없음
- **개선**: KPI 클릭 → 상세 드릴다운, 오늘의 인사이트 섹션 추가

### 3.3 스크롤/여백 기술 분석

**스크롤 문제 원인**:
- `body { height: 100% }` + `#__next { height: 100% }` 고정
- AppShell main에 `overflow-y-auto` 적용되나, paddingBottom이 탭바 높이만큼만 설정

**해결**:
```css
main {
  height: calc(100vh - 3.5rem - 4rem); /* statusbar + tabbar */
  overflow-y: auto;
  -webkit-overflow-scrolling: touch;
}
```

**여백 문제 원인**:
- 각 페이지 콘텐츠에 max-w 미설정 (consultation만 `max-w-lg mx-auto` 적용)

**해결**: AppShell main에 `max-w-md mx-auto w-full` 추가

### 4.2 P0 기능 상세

#### 예약 정보 입력 (Reservation/Booking)

사장님이 카카오톡/네이버 예약에서 받은 정보를 앱에 입력하는 기능.

**데이터 모델**: `BookingRequest` — 고객명, 전화, 예약일시, 채널(kakao/naver/phone/walk_in), 요청사항, 참고이미지
**UI 위치**: 홈 화면 "예약 관리" 섹션 또는 별도 `/reservations` 페이지
**핵심 흐름**: 예약 카드 → "상담 시작" 버튼 → 예약 정보가 상담 store에 pre-fill

#### 고객 정보 연동 (Customer Pre-fill)

상담 시작 시 예약/기존 고객의 선호도/최근시술/회원권을 한눈에 보여주는 기능.

**현재**: `CustomerInfoForm`에 기존 고객 선택 시 이름/전화번호만 fill
**개선**: 선호도, 최근 시술, 회원권 잔여, 예약 요청사항까지 한번에 표시

### 4.3 P1 기능 상세

#### 단골 고객 시스템
- `Customer` 타입에 `isRegular`, `regularSince`, `visitFrequency` 추가
- 고객 상세에서 별 아이콘 토글로 단골 지정/해제
- 대시보드에 단골 KPI 카드, 고객 목록에 단골 필터

#### 회원권 관리
- `Membership` 타입: 총횟수, 사용횟수, 잔여횟수, 구매금액, 만료일, 상태
- 상담 완료 시 자동 차감
- 고객 상세에 잔여 횟수 뱃지, 설정에 회원권 관리 섹션

#### 이미지 첨부
- 타입 레벨에서 `imageUrls` 이미 존재하나 UI 없음
- 예약 입력 시 참고 이미지, 시술 완료 후 결과 사진 첨부
- 고객 상세 갤러리와 연동

#### 고객 미니 대시보드
- 고객별 시술 통계 시각화 (디자인 비율 도넛 차트, 핸드/풋 비율 바, 6개월 방문 추이)
- `CustomerStats` 계산 함수로 처리

#### 대시보드 드릴다운
- KPI 카드 클릭 → 바텀시트 상세:
  - "총 매출" → 매출/원가/이익/마진율
  - "고객 수" → 단골/신규 탭 + 고객 리스트
  - "총 상담" → 담당자별/서비스별 분해

### 4.4 구현 권장 순서

**Wave 1 (P0)**: 예약 정보 입력 + 고객 연동 → 핵심 워크플로우 시연 가능
**Wave 2 (P1 빠른 임팩트)**: 단골 시스템 + 대시보드 드릴다운 → 사장님 가치 증명
**Wave 3 (P1 완성도)**: 회원권 관리 + 이미지 첨부 + 고객 미니 대시보드
**Wave 4 (P2)**: 프로 요금제 UI + 매장 휴무일 설정

### 5.2 온보딩 IA 재설계

**변경 후 순서** (8단계):
```
시작 → 매장정보 → 서비스 → 기본가격 → 추가금 → 시술시간 → 테마 → 완료
```

### 5.3 상담 최종 확인 섹션 순서 재배치

**현재**: 기본정보 → 고객정보 → 시술조건 → 디자인 → 추가옵션 → 가격상세

**변경 후** (클라이언트 요청):
```
[상단 고정] 최종 금액 (text-4xl) + 예상 시간 (text-xl) — 히어로 카드
1. 가격 상세 (접었다 펼치기)
2. 디자인
3. 시술 조건
4. 추가 옵션
5. 고객 정보 (마지막)
```

### 5.4 고객 상세 섹션 재배치

**현재**: 기본정보 → 태그 → 스몰토크 → 갤러리 → 이력 → 선호도

**변경 후**:
```
1. 기본정보 (미니 대시보드 + 회원권)
2. 이미지 갤러리 (시각적 첫인상)
3. 시술 이력 타임라인
4. 시술 성향 태그 (압축)
5. 선호도 프로필
6. 스몰토크 기록 (메모 성격)
```

### 5.5 대시보드 재구성

```
KPI: 오늘 매출 / 이달 매출 (기간 토글)
     예약 건수 / 평균 객단가 / 신규 고객 수

차트:
1. 매출 추이 (주간/월간)
2. 담당자별 매출 실적
3. 인기 서비스 차트
4. 고객 분석 (재방문율, 신규 비율)

신규 — 오늘의 인사이트:
• "그라데이션 요청 +40% 증가"
• "효주 선생님 매출 1위"
• "재방문율 68% (지난달 대비 +3%)"
```

### 5.6 설정 페이지 탭 기반 구조

**현재**: 단일 스크롤 페이지 (매장 → 선생님 → 서비스 → 테마 → 앱)

**변경**: 탭 기반 — `[매장] [서비스] [예약관리] [테마] [앱]`
