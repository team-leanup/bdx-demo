## 2. 인증 & 온보딩

### 2-1. 루트 리다이렉터 (`/`)

`useAuthStore`의 `isInitialized` 상태를 감시하며, 초기화 완료 시 조건 분기:

| 조건 | 이동 경로 |
|------|----------|
| `pendingGoogleSignup` 존재 | `/signup/google` |
| 로그인 + 온보딩 완료 | `/home` |
| 로그인 + 온보딩 미완 | `/onboarding` |
| 미로그인 | `/splash` |

### 2-2. 스플래시 & 인트로

```
앱 실행
  ↓
/splash — BDX 심볼 로고 + 도트 로딩 애니메이션 (5초)
  ↓  (자동 이동)
/intro — 앱 소개 슬라이드 (2장 스와이프)
  ↓
/login — 이메일 로그인 / Google OAuth
  └── /signup — 이메일 회원가입 (이용약관 동의)
      └── /signup/google — Google OAuth 후처리
```

### 2-3. 온보딩 (7단계)

최초 로그인 후 1회 진행. 프로그레스 바로 진행률 표시.

| 스텝 | 경로 | 입력 내용 |
|------|------|----------|
| 1 | `/onboarding/shop-info` | 샵 이름, 연락처 |
| 2 | `/onboarding/portfolio-upload` | 대표 디자인 사진 업로드 |
| 3 | `/onboarding/portfolio-classify` | 스타일 분류 + 대표 디자인 선택 |
| 4 | `/onboarding/pricing` | 카테고리별 가격/시간 설정 |
| 5 | `/onboarding/surcharges` | 제거/연장/추가옵션 |
| 6 | `/onboarding/notice` | 고객 안내 문구 |
| 완료 | `/onboarding/complete` | 설정 완료 → 현장모드 시작 |

### 주요 기능

- **예약 실시간 갱신**: 30초 폴링 + 브라우저 visibility change 이벤트
- **QR 모달**: 고객용 사전 상담 링크 QR 코드 생성
- **사전 상담 알림**: 고객이 Customer Link로 사전 상담을 완료하면 알림 표시
- **상담 이어하기**: 진행 중 상담이 있으면 Resume Dialog (이어하기/새로 시작)

| 전화 | 파란색 |

## 4. 상담 플로우

> 경로: `/consultation/*`
> 언어: 선택 언어 + 한국어 보조 (Dual Language)
> 핵심 기능: 외국인 고객과의 네일 시술 상담

### 4-1. 진입 방식 (3가지)

| 진입점 | `entryPoint` 값 | 스텝 수 | 설명 |
|--------|----------------|---------|------|
| 사장님이 직접 시작 | `staff` | 5 스텝 | 전체 상담 플로우 |
| 재방문 예약 | `return_visit` | 3 스텝 | 시작 → 고객 정보 → 최종 확인 (단축) |
| 고객 QR 링크 | `customer_link` | 5 스텝 | 고객 자가 입력 모드 |

**Staff 모드 vs Customer Link 모드 차이:**

| 항목 | Staff | Customer Link |
|------|-------|---------------|
| 기존 고객 검색 | ✅ 자동완성 | ❌ 비활성 |
| 이미지 업로드 위치 | 하단 | 최상단 (강조) |
| 무드 태그 | 하단 배치 | 상단 강조 |

| 예약 연동 시 | 이름/전화 자동 채움 + 파란 안내 배너 | — |

**검증:** 이름 필수. 비어있으면 에러 메시지 + 진행 차단.

### 4-4. Step 2 — 기본 조건 (`/consultation/step1`)

> stepNumber: 2/5

4개 섹션이 구분선으로 나뉘어 세로 배치.

| 섹션 | 컴포넌트 | 선택지 |
|------|---------|--------|
| 시술 부위 | `BodyPartSelector` | 손(hand) / 발(foot) |
| 오프 | `OffSelector` | 오프 없음 / 자샵오프 / 타샵오프 |
| 연장 | `ExtensionSelector` | 없음 / 리페어 / 연장 |
| 네일 쉐입 | `ShapeSelector` | 라운드 / 오발 / 스퀘어 / 스퀘오발 / 아몬드 / 스틸레토 / 코핀 |

**네일 쉐입 UI:** 각 쉐입의 SVG 일러스트 + 이름 레이블. 선택 시 테두리 강조.

**보호:** `useConsultationGuard()` 훅으로 Step 1을 건너뛰고 직접 접근 차단.

## 11. 상태 관리 & 데이터 흐름

### Zustand 스토어 맵

| 스토어 | 영속 방식 | 핵심 상태 |
|--------|---------|----------|
| `consultation-store` | sessionStorage | 현재 상담 전체 데이터, currentStep, entryPoint |
| `locale-store` | localStorage (previousLocale) | 현재 언어, 이전 언어 (복원용) |
| `auth-store` | persist | role, currentShopId, activeDesignerId, 온보딩 완료 |
| `app-store` | persist | shopSettings (가격표, 영업시간) |
| `shop-store` | persist | shop 정보, designers 배열 |
| `customer-store` | persist + DB 동기화 | customers[], 태그, 스몰토크 노트 |
| `records-store` | persist + DB 동기화 | ConsultationRecord[] |
| `reservation-store` | persist + DB 폴링(30초) | BookingRequest[] |
| `portfolio-store` | persist + DB | 포트폴리오 사진 |
| `parts-store` | persist | 커스텀 파츠 정의 |
| `onboarding-store` | persist | 온보딩 진행 상태 |

### 핵심 데이터 흐름: 상담 저장

```
handleSave() (summary 페이지)
  │
  ├─ 1. 신규 고객 → customer-store.createCustomer()
  │                → lib/db.dbUpsertCustomer() (Supabase)
  │
  ├─ 2. 상담 기록 → records-store.addRecord()
  │
  ├─ 3. 이미지 → portfolio-store.addPhoto() (base64인 경우)
  │
  ├─ 4. 고객 갱신 → customer-store.updateCustomer()
  │                 (방문횟수, 총매출, 시술이력)
  │
  ├─ 5. 메모 → customer-store.appendSmallTalkNote()
  │
  ├─ 6. 태그 → customer-store.updateTags()
  │             (특성 태그 → 고객 태그 자동 등록)
  │
  ├─ 7. 언어 → locale-store.restoreLocale()
  │
  └─ 8. 라우팅
       ├─ staff → /consultation/treatment-sheet
       └─ customer_link → /consultation/save-complete?mode=preconsultation
```

### 핵심 데이터 흐름: 예약 → 상담 시작

```
/home (예약 카드 "상담 시작" 클릭)
  │
  ├─ hydrateConsultation({
  │     ...booking.preConsultationData,
  │     bookingId,
  │     entryPoint: 'staff' | 'return_visit'
  │  })
  │
  └─ router.push('/consultation')
       → consultation store에 예약 데이터 사전 로드
       → 고객 정보 자동 채움
```

## 12. 다국어 시스템

### 지원 언어

| 코드 | 언어 | 용도 |
|------|------|------|
| `ko` | 한국어 | 기본, (main) 고정 |
| `en` | 영어 | 상담 시 선택 |
| `zh` | 중국어 | 상담 시 선택 |
| `ja` | 일본어 | 상담 시 선택 |

### 3가지 훅

| 훅 | 반환값 | 용도 |
|----|--------|------|
| `useT()` | `t` 함수 | 현재 선택 언어로 번역 |
| `useKo()` | `tKo` 함수 | 항상 한국어 번역 |
| `useLocale()` | `locale` 코드 | 현재 언어 코드 |

### Dual Language 패턴 (상담 페이지 전체 적용)

```tsx
const t = useT();        // 함수 직접 반환 (destructuring 금지)
const tKo = useKo();
const locale = useLocale();

// 외국어 큰 글씨 + 한국어 보조 텍스트
<span className="text-lg font-black">{t('common.next')}</span>
{locale !== 'ko' && (
  <span className="text-xs text-text-muted opacity-60">
    {tKo('common.next')}
  </span>
)}
```

### 언어 전환 생명주기

```
1. setConsultationLocale(lang)
   → 현재 locale을 previousLocale에 백업
   → 새 언어로 전환

2. 상담 진행 중
   → consultation/* 페이지에서 선택 언어 + 한국어 보조

3. 상담 완료 → restoreLocale()
   → previousLocale → locale 복원
   → (main) 페이지에서도 useLayoutEffect로 ko 강제)
```

### 페이지별 언어 정책

| 영역 | 정책 |
|------|------|
| `(main)/*` | 항상 한국어 (`layout.tsx`에서 `setLocale('ko')` 강제) |
| `consultation/*` | 선택 언어 + 한국어 보조 (Dual Language) |
| `(auth)/*` | 한국어 |
| `onboarding/*` | 한국어 |

### SVG 내 다국어 처리

SVG `<text>`는 동적 크기 조절 불가 → HTML 오버레이 뱃지로 처리 (OffSelector 참고)

### 13-2. 상담 플로우 — 스텝 네비게이션

```
[URL 라우팅] + [Zustand currentStep] 이중 동기화

goNext() / goPrev()
  → STEP_ORDER 배열 기반 인덱스 이동
  → setStep() + router.push() 동시 호출
```

**entryPoint별 스텝 순서:**

| entryPoint | 스텝 순서 |
|------------|----------|
| `staff` / `customer_link` | START → CUSTOMER_INFO → STEP1_BASIC → STEP2_DESIGN → TRAITS → SUMMARY |
| `return_visit` | START → CUSTOMER_INFO → SUMMARY (3스텝 단축) |

**가드:** `useConsultationGuard()` — 올바른 step이 아니면 진입 차단 (URL 직접 접근 방지)

### 13-3. 상담 시작 진입점

| 진입점 | 경로 |
|--------|------|

| 고객 상세 → 재예약 | hydrate → `/consultation` |
