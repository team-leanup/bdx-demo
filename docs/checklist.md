## 2. 홈 화면 — 스케줄 중심으로 개편

> 핵심: 원장님이 앱 열면 **오늘 스케줄 + 고객 특징 태그**가 바로 보여야 함

### 2.1 🟢 홈 화면을 스케줄표 중심으로 변경
- **완료**: 스케줄표 중심 레이아웃 적용됨
- **완료 추가**: 기록 관리 페이지에서 `useShopStore((s) => s.getActiveDesigners())`처럼 새 배열을 반환하던 selector를 제거하고, 원본 `designers` 상태를 `useMemo`로 파생하도록 바꿔 React `getSnapshot` 무한 루프 경고를 해소함
- **완료 추가**: 홈 `오늘의 예약` 카드 내부에 예약 참고 이미지(`referenceImageUrls` / 사전상담 referenceImages)와 기존 고객 포트폴리오 사진을 작은 썸네일 스트립으로 노출하고, 예약 폼 업로드 이미지는 `blob:` 임시 URL 대신 data URL로 저장해 새로고침 후에도 썸네일이 깨지지 않게 함

### 2.2 🟢 예약 카드에 고객 특징 태그 표시
- **완료**: 예약 카드가 `customerId`로 고객의 주요 태그를 조회해 칩 형태로 최대 3개까지 표시함. pinned 태그가 있으면 우선 노출하고, 없으면 기존 고객 태그를 우선순위대로 보여줌
  ```
  김지영 · 자석젤
  [큐티클 민감] [오버레이 선호] [유지력 약함]
  ```
- **참고**: 시술 전 알림 모달은 기존처럼 pinned 태그만 사용하고, 카드 표시용 태그만 별도 우선순위 조회 로직으로 분리함
- **완료 추가**: `records` 일간 디자이너 캘린더의 드래그 예약 카드도 홈 카드처럼 서비스 라벨 + 고객 태그를 카드 내부에 표시하도록 보강하고, 1시간 슬롯 높이를 키워 태그 줄이 잘리지 않게 조정함

### 2.3 🟢 태그 색상 코딩
- **완료**: `CustomerTag.accent` 커스텀 색상을 유지하면서 카테고리별 기본 색상을 자동 적용하도록 공통 태그 컬러 시스템을 반영함. 스케줄 카드, 고객 상세, 예약 연결 모달, 시술 전 알림, 기록 상세에서 동일한 색상 규칙을 사용함

### 2.4 🟢 상담 준비 상태 아이콘
- **완료**: 예약별 사전 상담 완료 여부를 저장하고, 홈/예약 리스트/캘린더/예약 상세에서 `🟢 디자인 확정` / `🟡 현장 상담 필요` 상태 아이콘으로 표시함. 상담 링크 제출이 완료되면 자동으로 디자인 확정 상태로 전환됨

### 2.5 🟢 사전상담 알림 센터 실데이터 연동
- **완료**: 상단/사이드 알림 벨을 실제 사전상담 알림 센터와 연결하고, 미확인 개수를 배지로 표시함. 알림 목록은 예약 실데이터의 `preConsultationCompletedAt` 기준으로 생성되며, 읽음 상태는 예약별로 저장됨
- **완료 추가**: 홈 상단 배너와 알림 센터가 동일한 읽음 상태를 공유하고, 알림 클릭 시 `records?bookingId=`를 통해 해당 예약 상세 바텀시트가 자동으로 열리도록 연결함

### 2.6 🟢 오늘 날짜/집계 기준을 한국 시간으로 통일
- **완료**: 홈/대시보드/기록/캘린더에서 `today` 계산과 날짜 표기를 `Asia/Seoul` 기준으로 통일해, 서버가 UTC여도 한국 기준 오늘 날짜가 어제로 보이던 문제를 수정함
- **완료 추가**: 당일 예약/상담 집계도 KST 날짜 문자열로 비교하도록 맞춰, 자정 전후에도 `오늘` 카드와 KPI 값이 한국 시간 기준으로 일관되게 보이도록 정리함
- **완료 추가**: 예약/상담/고객/포트폴리오/설정 저장 시 생성되는 새 타임스탬프, 포트폴리오 기본 촬영일, mock 상대 날짜, 상대 날짜 계산까지 KST helper로 통일해 전체 흐름에서 날짜 기준이 섞이지 않게 정리함

### 2.7 🟢 홈 예약 카드 `고객 연결 필요` 버튼 좌측 정렬
- **완료**: 홈 `오늘의 예약` 카드에서 고객 미연결 상태일 때 보이는 `고객 연결 필요` 버튼에 `self-start text-left` 정렬을 적용해, 카드 내부에서 가운데로 보이던 CTA가 왼쪽 정보 컬럼 시작선에 맞춰 정렬되도록 수정함
- **완료 추가**: 홈 `오늘의 예약` 카드에서 포트폴리오/참고 썸네일이 생기면 모바일 한 줄 폭이 무너지던 문제를 수정하고, 모바일에서는 `시간 + 본문`과 `썸네일 + 액션`을 두 줄로 분리해 고객명/태그가 세로로 찌그러지지 않게 정리함
- **완료 추가**: 홈 카드 섹션 타이틀을 실제 사용 맥락에 맞춰 `오늘의 예약`에서 `오늘의 상담`으로 변경해, 상단 CTA/기록 흐름과 용어가 자연스럽게 이어지도록 정리함
- **완료 추가**: 홈 통계 카드의 `오늘 예약`도 `상담 예정`으로 맞추고, 온보딩 가이드/인트로 데모에 남아 있던 `오늘의 예약` 문구도 `오늘의 상담` 기준으로 통일해 홈 진입 문맥 전체를 일관되게 정리함
- **완료 추가**: 홈 Hero CTA의 보조 카드 제목도 `새 예약 등록`에서 `새 상담 등록`으로 바꿔, 홈 진입 주요 액션의 용어를 상담 중심 톤으로 맞춤

### 3.2 🟢 사진 상세에서 시술 정보 + 가격 표시
- **완료**: 사진 상세 페이지에서 시술 정보 + 가격 표시 구현됨

### 3.3 🟢 포트폴리오 ↔ 고객 ↔ 상담 데이터 연결
- **완료**: 포트폴리오-고객-상담 간 데이터 연결 구현됨

- **완료 추가**: 선택한 포트폴리오 이미지는 상담 `referenceImages`로만 미리 채우고, 고객 정보는 비워 둔 상태로 `/consultation/customer`에서 새 상담을 시작하게 정리함

### 4.6 🟢 고객 목록 검색 + 페이지네이션
- **완료**: 고객 목록 검색 범위를 이름/전화번호/담당자에서 고객 ID와 태그까지 확장하고, 검색 placeholder도 실제 검색 범위에 맞게 갱신함
- **완료 추가**: 고객 목록에 10명 단위 페이지네이션과 `이전/다음` 이동 UI를 추가하고, 검색어나 필터가 바뀌면 자동으로 1페이지로 리셋되게 정리함

### 4.7 🟢 고객 상세 → 상담 기록 탭 연동 강화

- **완료 추가**: 기록 페이지 상단에 고객 기준 필터 배너와 `필터 해제` 액션을 추가해, 특정 고객 기록만 보는 상태를 명확히 표시함
- **완료 추가**: 고객 상세의 시술 사진 그리드 삭제 버튼이 `group-hover`에만 묶여 있던 문제를 수정해, 태블릿/모바일에서도 X 버튼이 항상 보여 바로 삭제할 수 있고 데스크톱에서는 기존 hover 감도도 유지되게 조정함
- **완료 추가**: 고객 상세의 최근 시술/시술 사진 갤러리에서 빈 `imageDataUrl` 항목을 렌더 전에 제외해, `next/image`가 빈 `src`로 콘솔 에러를 내지 않도록 안전 가드를 추가함

## 6. 상담 UX — 6단계로 간소화

> 핵심: 현재 상담이 복잡함. **최대한 빠르게 끝나야** 시술 가능 고객 수가 늘어남

### 6.1 🟢 상담 흐름 재구성
- **완료**: 상담 6단계 흐름 구현됨
- **완료 추가**: `/consultation` 시작 화면의 히어로 영역은 BDX 세로 로고 중심으로 정리하고, 상단 심볼 배지는 제거해 더 단정한 랜딩 구성을 유지함
- **완료 추가**: 메인 상담 흐름을 `step1 → step2 → 특성 선택 → 고객 정보 → 요약` 순서로 재정렬해, 고객 정보 입력이 특성 선택 다음 단계에서 나오도록 이동함
- **완료 추가**: 홈 `오늘의 예약`, 예약 캘린더, 고객 상세에서 `상담 시작` 시 기존 step4(customer)로 점프하지 않고 항상 `step1`부터 시작하도록 진입 라우팅과 초기 상태를 정리함
- **완료 추가**: 예약에 이미 연결된 고객이 있거나 고객 정보 단계에서 `기존 고객`을 선택한 경우, 다음 단계 이동 시 `customerId`를 유지하도록 수정해 요약 저장 시 같은 고객이 중복 생성되지 않게 함
- **완료 추가**: customer link 상담 저장 완료 후에는 `/consultation/save-complete` 대신 바로 `/home`으로 이동하도록 변경해, 저장 직후 다시 상담 플로우로 돌아오는 느낌을 없앰

### 6.2 🟢 기존 라우트 → 신규 라우트 마이그레이션
- **완료**: consultation/ 라우트 구조 전환됨
