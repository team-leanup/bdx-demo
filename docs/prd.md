## 1. 제품 개요

### 1.1 서비스 정의

**BDX(Beauty Decision eXperience)**는 네일샵 현장에서 고객 상담 과정을 구조화·문서화하고, 가격과 시간을 자동 산출하며, **고객 관리와 매장 데이터 분석**을 통해 네일샵 운영 전반을 지원하는 **네일 상담 & 고객 관리 플랫폼**입니다.

기존 수기 메모나 구두 합의에 의존하던 네일 상담 프로세스를 디지털화하여, 상담 → 합의 → 기록 → 고객관리 → 분석의 전 과정을 하나의 플랫폼에서 처리합니다.

### 1.2 핵심 가치 (우선순위순)

**1순위: 마이페이지 — 고객 관리 (Customer Management)**

네일샵의 핵심 자산은 "고객 데이터"입니다. BDX는 고객별 시술 이력, 선호도, 시술 중 나눈 스몰토크 내용까지 빠짐없이 기록하여, 재방문 시 "지난번에 OO 이야기 하셨죠?" 수준의 세심한 단골 관리를 가능하게 합니다. 고객 한 명을 클릭하면 성명, 전화번호, 담당 선생님, 시술 이미지, 시술 정보, 대화 기록, 시술 성향 키워드까지 한눈에 확인할 수 있습니다.

**2순위: 대시보드 — 매장 데이터 관리 (Shop Analytics)**

상담 데이터가 쌓이면 매장 전체의 매출 추이, 인기 서비스, 고객 패턴, 디자이너 성과 등을 자동으로 분석합니다. 사장님이 "감"이 아니라 "데이터"로 매장을 운영할 수 있도록 합니다.

**3순위: 상담 흐름 — 시술 합의 (Consultation Flow)**

업주가 태블릿에서 고객과 함께 선택지를 눌러가며 상담을 진행하면, 시스템이 실시간으로 가격과 시간을 계산하고 합의 내용을 자동 문서화합니다. 손가락 캔버스에서 파츠 위치까지 기록할 수 있습니다.

### 1.3 타겟 사용자

| 사용자 | 설명 | 주요 니즈 |
|--------|------|-----------|
| **네일샵 원장/사장님** | 매장을 운영하며 상담·시술을 직접 담당 | 단골 관리, 매장 데이터 분석, 상담 표준화 |
| **네일 아티스트(선생님)** | 원장 소속으로 시술을 담당하는 직원 | 고객 히스토리 확인, 빠른 상담 기록 |
| **네일샵 고객** | 시술을 받는 소비자 (외국인 포함) | 시술 내용·가격의 명확한 이해 |

### 1.4 디바이스 전략 — 앱처럼 보이는 웹

**Tablet-First, App-Like Web** — 네일샵 현장에서 태블릿 위에서 고객과 함께 상담하는 것이 핵심 시나리오입니다. 전통적인 웹페이지 형태가 아니라, **네이티브 앱처럼 보이고 동작하는 웹 애플리케이션**으로 설계합니다.

| 디자인 원칙 | 적용 방법 |
|-------------|-----------|
| **하단 탭 네비게이션** | 웹 상단 메뉴바 대신 모바일/태블릿 앱 스타일의 하단 탭 바 사용 |
| **풀스크린 레이아웃** | 브라우저 주소창을 제외한 전체 화면을 앱처럼 활용 (PWA 지원) |
| **스와이프/제스처** | 카드 스와이프, 풀 투 리프레시 등 네이티브 앱 인터랙션 적용 |
| **트랜지션 애니메이션** | 페이지 전환 시 슬라이드/페이드 등 앱 스타일 전환 효과 |
| **둥근 모서리 + 카드 UI** | 네일샵 감성에 맞는 부드러운 라운드 카드 기반 레이아웃 |
| **상태바 영역** | 상단에 매장명, 현재 시간, 알림 아이콘 등 앱 스타일 헤더 |

## 2. 디자인 시스템 — 테마 커스터마이징

### 2.1 테마 시스템 개요

BDX의 핵심 차별점 중 하나는 **사장님이 자신의 네일샵 브랜드 이미지에 맞게 메인 컬러를 직접 선택**할 수 있는 테마 시스템입니다. 온보딩 과정 또는 설정에서 프리셋 테마를 선택하면 전체 UI가 해당 컬러로 통일됩니다.

### 2.2 프리셋 테마 (8종)

| 테마 | 메인 컬러 | HEX | 서브 컬러 | 분위기 | 어울리는 매장 |
|------|-----------|-----|-----------|--------|-------------|
| **로즈 핑크** (기본) | 핑크 | `#F472B6` | `#FDF2F8` | 사랑스럽고 여성스러운 | 트렌디한 네일샵, 20~30대 타겟 |
| **소프트 아이보리** | 아이보리 | `#F5F0E8` | `#FFFDF7` | 따뜻하고 자연스러운 | 감성 네일샵, 미니멀 인테리어 |
| **웜 베이지** | 베이지 | `#D4A574` | `#FFF8F0` | 고급스럽고 차분한 | 프리미엄 네일샵, 고급 인테리어 |
| **모던 블랙** | 블랙 | `#1A1A2E` | `#16213E` | 세련되고 모던한 | 시크한 네일바, 남성/유니섹스 |

### 2.3 테마 구성 요소

각 테마는 다음 요소에 일괄 적용됩니다:

| 요소 | 적용 범위 |
|------|-----------|
| **메인 컬러** | CTA 버튼, 선택 상태 하이라이트, 프로그레스 바, 탭 활성 상태 |
| **서브 컬러 (배경)** | 페이지 배경, 카드 배경, 섹션 구분 |
| **텍스트 컬러** | 기본 다크 그레이 (#1F2937) 고정, 강조 텍스트에 메인 컬러 적용 |
| **아이콘/일러스트** | 메인 컬러 톤으로 렌더링 |
| **차트/그래프** | 대시보드 차트 색상에 메인 컬러 그라데이션 적용 |

## 3. 인증 시스템

### 3.1 로그인 방식 (2가지)

BDX는 **구글 로그인**과 **이메일/패스워드 로그인** 두 가지 방식을 모두 지원합니다.

| 방식 | 구현 | 비고 |
|------|------|------|
| **구글 로그인** | Supabase Auth (Google OAuth) | 원터치 로그인, 추가 정보 입력 불필요 |
| **이메일/패스워드** | Supabase Auth (Email/Password) | 이메일 인증 + 비밀번호 설정, 비밀번호 찾기 지원 |

### 5.2 손가락 캔버스 기능 상세 (★ 신규)

#### BDX-CNV-001 손가락 캔버스 메인
손 모양 일러스트 위에서 각 손가락별로 시술 내용을 시각적으로 기록합니다. 왼손/오른손 전환이 가능하며, 핸드/풋에 따라 손/발 일러스트가 변경됩니다.

#### BDX-CNV-002 파츠 위치 마킹
각 손가락 위에 파츠를 드래그&드롭으로 배치합니다. 파츠 종류(스톤, 참, 호일, 글리터 등)와 등급(1000/2000/3000원)을 선택할 수 있으며, 배치된 위치가 시각적으로 표시됩니다.

#### BDX-CNV-003 포인트 손가락 지정
포인트 아트가 적용될 손가락을 터치로 지정합니다. 지정된 손가락은 하이라이트 표시되며, 포인트 수에 따라 가격이 자동 재계산됩니다.

### 5.3 마이페이지 기능 상세 (★ 최고 우선순위)

#### BDX-MYP-007 고객 상세 — 기본 정보
고객 성명, 전화번호, 담당 선생님, 첫 방문일, 총 방문 횟수, 평균 결제 금액을 한눈에 표시합니다.

#### BDX-MYP-008 고객 상세 — 시술 성향 키워드 태그
고객의 시술 성향을 키워드 기반 드롭다운(복수 선택 가능)으로 관리합니다. 디자인, 쉐입, 길이, 표현, 파츠, 컬러, 기타 카테고리별 프리셋 태그가 제공되며, 직접 입력으로 커스텀 태그도 추가 가능합니다.

#### BDX-MYP-009 고객 상세 — 스몰토크 기록
시술 중 고객과 나눈 대화 내용을 날짜별로 자유 텍스트로 기록합니다. "결혼 준비 중", "일본 여행 다녀옴", "건조한 손 관련 상담" 등 재방문 시 자연스러운 대화를 이어갈 수 있는 핵심 정보입니다.

#### BDX-MYP-010 고객 상세 — 시술 이미지 갤러리
각 시술 기록의 완성 사진과, 고객이 제공한 아트 레퍼런스 사진을 시간순으로 갤러리 형태로 표시합니다. 고객 정보 입력 시 기존 이미지가 자동 연동됩니다.

#### BDX-MYP-011 고객 상세 — 시술 이력 타임라인
방문일, 시술 부위, 디자인 범위, 금액, 담당 선생님을 시간순 타임라인으로 표시합니다.

#### BDX-MYP-012 고객 상세 — 선호도 프로필
체크리스트 데이터를 기반으로 자동 축적된 선호 쉐입, 길이, 두께, 큐티클 민감도, 메모입니다. 재방문 시 체크리스트에 자동 반영됩니다.

### 5.4 Scope 외 (Out of Scope)

| 항목 | 사유 |
|------|------|
| 예약/결제/정산 시스템 | PG 연동 필요, 예약금은 "기록용"으로만 운영 |
| 운영자(플랫폼) 전용 어드민 | MVP 단계에서 불필요 |
| 앱(WebView/네이티브) 전환 | PWA로 앱 경험 제공, 추후 검토 |
| 레퍼런스 이미지 별도 단계 | 고객 정보 연동으로 대체 |

## 6. 데이터 모델

### 6.1 핵심 엔티티 (11개 — 2개 추가)

| 엔티티 | 설명 | 주요 관계 | 변경사항 |
|--------|------|-----------|----------|
| Owner | 업주 계정 (Google/Email+PW) | 1:1 → Shop | 인증 방식 확장 |
| Shop | 네일샵 정보 + 기본 설정 + **테마** | 1:N → Designer, Customer | theme_id 추가 |
| Designer | 선생님(디자이너) 정보 | N:1 → Shop | - |
| Customer | 고객 정보 (전화번호 기준 식별) | 1:N → ConsultationRecord | - |
| CustomerPreference | 고객 선호 프로필 | 1:1 → Customer | - |
| **CustomerTag** | 고객 시술 성향 키워드 태그 | N:N → Customer | ★ 신규 |
| **SmallTalkNote** | 고객 스몰토크 기록 | N:1 → Customer | ★ 신규 |
| ConsultationRecord | 상담/시술 기록 | N:1 → Customer, Shop, Designer | canvas_data 추가 |
| Checklist | 시술 당일 체크리스트 | 1:1 → ConsultationRecord | - |
| ArtImage | 시술/레퍼런스 이미지 | N:1 → ConsultationRecord, Customer | 이름 변경 |
| CanvasData | 손가락 캔버스 데이터 (JSON) | 1:1 → ConsultationRecord | ★ 신규 |

### 6.2 신규 엔티티 상세

**CustomerTag (고객 성향 태그)**
```
id (PK)
customer_id (FK)
tag_category: enum (design/shape/length/expression/parts/color/etc)
tag_value: string
is_custom: boolean (프리셋 vs 직접입력)
created_at: timestamp
```

**SmallTalkNote (스몰토크 기록)**
```
id (PK)
customer_id (FK)
consultation_record_id (FK, nullable)
note_text: text
created_at: timestamp
created_by: designer_id (FK)
```

**CanvasData (캔버스 데이터)**
```
id (PK)
consultation_record_id (FK)
hand_type: enum (left_hand/right_hand/left_foot/right_foot)
finger_data: JSON  // 각 손가락별 파츠 위치, 아트 유형, 컬러 정보
created_at: timestamp
```
