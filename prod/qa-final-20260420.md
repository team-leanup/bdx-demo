## 3️⃣ 보안 이슈 (CRITICAL 3 · HIGH 4 · MED 3 · LOW 2)

### 🔴 CRITICAL

#### C1. PIN 해시가 클라이언트 localStorage에 저장
- **위치**: `src/store/auth-store.ts:499-517, L464-489`
- **문제**: `passwords` 객체 전체가 Zustand persist에 포함 → localStorage `bdx-auth` 키에 PBKDF2 해시 저장. 고정 salt(`bdx-pin-salt-v2:${designerId}`) + 4자리 PIN = GPU 오프라인 크래킹으로 초단위 추출 가능. **기본 PIN `1234` 하드코딩**으로 신규 디자이너 즉시 접근 가능.
- **조치**: `designers.pin_hash` 컬럼 서버 저장, 최초 로그인 시 PIN 변경 강제.

#### C2. 데모 쿠키 클라이언트측 우회 가능
- **위치**: `auth-store.ts:164-177`, `middleware.ts:53`
- **문제**: `document.cookie = 'bdx-demo=true'` 주입만으로 Supabase 세션 없이 `role:'owner'` + `shop-demo` 접근. 미들웨어도 동일 쿠키 체크로 인증 검사 bypass.
- **조치**: 환경 변수 `NEXT_PUBLIC_DEMO_ENABLED`로 production 기본 비활성화, 또는 IP/시간제 토큰.

#### C3. Storage 업로드 MIME 검증 누락
- **위치**: `db.ts:1684-1711, L1207-1216`, `PhotoCapture.tsx`
- **문제**: `dataUrlToBlob`이 data URL 헤더의 MIME을 신뢰. `portfolio-images` / `designer-profile-images` 버킷이 public이므로 `image/svg+xml` 위장 XSS 페이로드 업로드 → Stored XSS 가능.
- **조치**: 화이트리스트(`['image/jpeg','image/png','image/webp','image/gif']`) 검증 필수.

### 🟠 HIGH

#### H1. `create_shop_account` RPC `p_user_id` 주입
- **위치**: `db.ts:224-228`
- **문제**: 클라이언트가 임의 userId 전달. DB 함수 내 `auth.uid()` 사용하도록 수정 필요.

#### H2. Rate Limiting 완전 부재
- **위치**: `/pre-consult/[shopId]/confirm` 공개 엔드포인트, 로그인
- **문제**: 공개 예약 제출 무제한 호출 가능 → 스팸 예약 생성 위험.
- **조치**: Upstash Ratelimit + Vercel Edge Middleware (IP당 분 5회 등).

#### H3. CSP `unsafe-inline`
- **위치**: `middleware.ts:25`, `layout.tsx:77`
- **문제**: 테마 부트스트랩 inline 스크립트 때문에 `unsafe-inline` 유지 → XSS 방어 약화.
- **조치**: `nonce` 기반 CSP 전환.

#### H4. `createId` 예측 가능 (IDOR)
- **위치**: `db.ts:46-48`
- **문제**: `${prefix}-${Date.now()}-${random 6자}` → 같은 샵 내 booking/consultation_link ID 열거 가능.
- **조치**: `crypto.randomUUID()` 사용.

### 🟡 MED/LOW
- MED: `dangerouslySetInnerHTML` 경계 관리, 전화번호 형식 검증 부재, PBKDF2 salt 하드코딩
- LOW: `next.config.ts` 헤더 fallback 없음, `/share` 공개 경로 범위 재검토

### Supabase Security Advisor 5건 WARN
- `touch_consultation_links_updated_at` 함수 search_path 미고정
- Storage 버킷 3개(designer-profile-images, portfolio-images, pre-consult-refs) public listing 허용

## 5️⃣ UI/UX 이슈 (모바일, CRITICAL 2)

### 🔴 CRITICAL

#### U1. `text-[10px]` 221건 남용 (82개 파일)
- **문제**: 10px은 iOS 최소 렌더 크기(11px) 이하로 가독성 심각. 네일샵 사장님 고령층 대상 주요 불만 예상. `pre-consult/complete/page.tsx:91,105,132` 등 **고객 대면 화면**에서도 적용됨.
- **조치**: 보조 텍스트 `text-xs`(12px) 최솟값, 일반 레이블 `text-sm`(14px) 이상.

#### U2. `prefers-reduced-motion` 전무
- **문제**: 체크마크 spring, 홈 stagger 0.08s, 페이지 진입 y-offset 등 → 전정기관 민감 사용자 멀미 유발 + 접근성 법적 요건.
- **조치**: Framer Motion `useReducedMotion()` 훅 적용.

### 🟠 HIGH (Top 5)
1. **BottomTabBar 6탭 과밀** — iOS HIG 5개 권장, 가로 60px 터치 타겟 미달, 레이블 truncate 발생
2. **고객 목록 필터/페이지네이션 버튼 24~26px** — WCAG 44px 미달, 시술 중 오조작 빈발 예상
3. **고객 카드 `aria-label` 부재** — 화면읽기 지원 약함
4. **완료 화면 1초 딜레이** — 체크마크 0.5s + 제목 0.7s + 슬롯 0.9s → 사용자 재제출 유도 위험
5. **홈 화면 7개 섹션 과밀** — fold above 핵심 CTA만 유지 권장

### ✅ 잘 된 점

- safe-area 처리 체계화 (`.safe-bottom` 유틸 + `env(safe-area-inset-bottom)`)
