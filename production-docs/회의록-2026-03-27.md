### 이슈 2: 3 step에서 빠지는 정보

- **현재 수집 항목**: bodyPart, offType, extensionType, nailShape, designScope, expressions, canvasData, traits
- **3 step 축소 시 빠질 가능성**: nailShape(현장 결정), designScope(포트폴리오에서 추론), traits(optional), canvasData(A-10으로 제거)
- **빠지면 안 되는 것**: offType(가격 영향), extensionType(가격 영향), bodyPart(시술 범위)
- **상태**: **B-1에서 Jee가 정의해야 함** — 이 정의 없이는 A-2 구현 불가

### 이슈 3: 포트폴리오 통합 후 staff/customer_link 차이

- **통합해도 유지해야 할 차이**:
  - 디자이너 선택: staff 모드에서만 (고객은 디자이너 선택 불가)
  - 언어 선택: staff 모드에서만 (customer_link는 링크 생성 시 설정)
  - 고객 정보 prefill: customer_link에서만 (예약 등록 시 입력된 정보)
- **결론**: `entryPoint` 자체는 존속 필요. A-4는 PortfolioBrowser 조건만 제거

### 이슈 4: 캔버스 보조화 후 canvasData 처리

- **STEP_ORDER에서 제거** → 별도 접근 경로 필요
- **옵션**: treatment-sheet와 통합, 또는 상담 상세 페이지 내 별도 버튼
- **기존 저장된 canvasData**: 그대로 유지 (마이그레이션 불필요)
- **return_visit 플로우**: `RETURN_VISIT_STEP_ORDER`는 이미 캔버스 미포함 (`START → CUSTOMER_INFO → SUMMARY`)
