## 1. 핵심 결정사항 (확정)

| 항목 | 결정 | 비고 |
|------|------|------|
| 결제 기능 | 기록만 (PG 연동 X) | 정산 화면 + DB 저장 |
| 기존 상담 플로우 | 대체 (현장모드로 교체) | `/consultation/*` 제거 후 새 플로우 |
| 이미지 저장소 | Supabase Storage | 업로드/조회 통일 |
| 공유카드 이미지 | 템플릿 + 시술 사진 오버레이 | 서버사이드 복잡한 생성 X |
| 카카오톡/네이버 연결 | 외부 링크 (SDK 연동 X) | 설정에 URL 입력 → 단순 링크 이동 |
| 시술 중 UI | 추가 버튼만 있는 미니 패널 | 별도 큰 화면 X |
| 사전 상담 링크 배포 | 복사 기능 → 사용자가 수동 전송 | 자동 발송 X |
| 가격 체계 | 옵션별 개별 설정 가능 | 카테고리 기본가 + 옵션별 추가가 각각 |

## 2. 문서별 요구사항 요약 + 주의사항

### 2-1. 공유카드 기능 (`공유카드-기능-최종-정의.md`)

**핵심**: 시술 결과 → 이미지 카드(SNS용) + 링크 카드(전환용) 동시 생성

**구현 포인트:**
- 이미지 카드: HTML/CSS 템플릿에 시술 사진 배치 → `html2canvas`로 이미지 캡처/다운로드. 시술 사진 + 최소 텍스트
- 링크 카드: 비로그인 퍼블릭 페이지. 시술 이미지 + 시술 요약 + 상담 CTA + 예약 연결(카카오/네이버/전화)
- 샵 정보(카카오 링크, 네이버 URL, 전화번호)는 설정에서 1회 입력 → 모든 카드에 자동 적용
- OG 메타태그 동적 생성 필요 (SNS 공유 미리보기)
- 이미지 다운로드 + 링크 복사 기능 제공

**주의:**
- 이미지에 가격/버튼/링크 포함 금지 (관심 유도 역할만)
- 링크 카드의 "상담하기" CTA → 새 현장모드 플로우로 연결 (디자인 정보 자동 입력)

### 2-2. 디자인 고르기 - 현장모드 (`디자인-고르기-현장모드-흐름도.md`)

**핵심**: 포트폴리오 기반 빠른 확정 도구. 상담이 아니라 결정을 끝내는 흐름.

**STEP별 구현:**

| STEP | 화면 | 핵심 |
|------|------|------|
| 1 | 포트폴리오 그리드 | 카테고리 필터(심플/프렌치/자석/아트) + 추천/인기 탭. 디자인 이미지 + 가격 표시 |
| 2 | 선택 디자인 크게 | 확정 느낌. "이 디자인 기준으로 정리" |
| 3 | 최소 옵션 선택 | 제거(없음/자샵/타샵), 길이(기본/짧게/연장→자연/중간/길게), 추가옵션(파츠/글리터/포인트). 3~5초 내 완료 |
| 4 | 실시간 요약 바 (하단 고정) | 디자인 + 옵션 + 예상 금액 + 예상 시간. 선택 즉시 반영 |
| 5 | 시술 진행 | 별도 화면 없음 |
| 6 | 시술 중 추가 | 미니 패널(파츠/글리터/포인트/연장/기타). 누르면 즉시 금액 반영. 고객에게 강제 노출 X |
| 7 | 최종 정산 | 디자인 + 기본옵션 + 추가항목 + 최종금액 + 총시간 |
| 8 | 결제 (기록) | 금액 확정 → DB 저장 |
| 9 | 고객정보 입력 (선택) | 이름/연락처. 강제 아님 |
| 10 | 사진 저장 | 촬영/갤러리 → Supabase Storage |
| 11 | 자동 데이터화 | 시술기록 + 고객카드 + 포트폴리오 자동 연결 |

**주의:**
- 기존 `/consultation/*` (step1~3 + canvas + summary) 완전 대체
- 추가 옵션은 "업셀링" 느낌 X → "디자인 완성도" 느낌으로 제안
- 고객정보는 확정 후 나중에 (흐름 끊지 않기)
- 태블릿 최적화 고려 필요

### 2-3. 미리 정하기 UX (`미리-정하기-UX-흐름도.md`)

**핵심**: 사전 링크로 고객이 혼자 미리 선택. 비로그인 퍼블릭 페이지.

**STEP별 구현:**

| STEP | 화면 | 핵심 |
|------|------|------|
| 0 | 시작 화면 | "미리 선택하면 매장에서 빠르게" 안내 |
| 1-1 | 카테고리 선택 | 심플/프렌치/자석/아트 |
| 1-2 | 디자인 이미지 선택 | 카테고리 내 대표 이미지. 선택 후 안심 문구 |
| 1-3 | 예상 가격대 노출 | 범위만 (확정 X). "40,000~70,000원" |
| 2-0 | 참고 이미지 업로드 (선택) | Supabase Storage. 건너뛰기 가능 |
| 2-1 | 필수 조건 | 제거 여부 → (분기) 자샵/타샵. 길이 → (분기) 연장 세부. 손톱 모양 |
| 2-2 | 디자인 느낌 | 심플/프렌치/트렌디/화려 |
| 2-3 | 고객 스타일 | 단일 선택 + 복수 선택. BDX 차별화 포인트 |
| 2-4 | 추가 옵션 | 접기/펼치기. 선택 시에만 추가금 노출 |
| 2-5 | (중간 피로 관리 문구) | 별도 화면 X, 흐름 속에 삽입 |
| 2-6 | 선택 내용 확인 | 전체 요약 + 수정/진행 버튼 |
| 3-1 | 최종 요약 | 디자인 + 옵션 + 소요시간 |
| 3-2 | 확정 금액 | 확정가 + "추가금 없이 진행" 안내 |
| 3-3 | 예약 직전 클로징 | "편하게 방문하시면 돼요" |
| 3-4 | 예약 정보 입력 | 이름/연락처 → DB 저장 |
| 4 | 완료 | "예약 완료 + 자동 저장" 안내 |

**주의:**
- 비로그인 퍼블릭 페이지 → 세션/상태 관리 방식 결정 필요 (localStorage 추천)
- 상담 중 가격을 계속 크게 보여주지 않음 (몰입 깨짐 방지)
- 분기형 질문: "기존 네일 있어요" → 제거 여부 질문 이어짐. "연장" → 길이 세부 이어짐
- 링크 생성 = 복사 기능. 사용자가 카톡 등으로 수동 전송
- 오프라인(매장)에서도 동일 플로우 사용 가능 (직원이 옆에서 함께 진행)

### 2-4. 시술후 결제 UX (`시술후-결제-UX-설계.md`)

**핵심**: 시술 전 = 기준 합의 (예상가), 시술 후 = 최종 정산 (확정가). 이 구분 고정.

**구현 포인트:**
- 시술 전 화면: "이 기준으로 진행" + 예상 금액/시간. 확정가 아님을 명시
- 시술 중: 직원 미니 패널로 추가사항 즉시 반영 (현장모드 STEP 6과 동일)
- 시술 후 정산 화면: 디자인 + 기본옵션 + 추가항목 + 최종금액. "추가 내용 반영된 최종 금액" 문구
- 결제 = DB에 금액 기록 (PG X)
- 결제 후 → 사진 저장 → 포트폴리오 자동 연결

**주의:**
- 정산 화면은 "계산 화면"이 아니라 "정리 화면" — 뭘 했고 왜 이 금액인지 보여줘야 함
- 고객정보: 기존 고객 = 입력 없음, 신규 워크인 = 결제 직전/직후에 선택적
- 사진 저장: "오늘 시술 기록으로 남겨둘게요" — 루틴처럼 느끼게
- 모든 시술 사진 ≠ 포트폴리오 대표 사진. 기본값은 "기록 + 포트폴리오 저장", 대표 노출은 관리화면에서 별도 설정

### 2-5. 온보딩 (`온보딩-최종-흐름도.md`)

**핵심**: 3분 내 완료. 행동 설계 기반. 기존 `/onboarding/*` 대체.

**STEP별 구현:**

| STEP | 화면 | 핵심 |
|------|------|------|
| START | 시작 | "3분만에 준비" |
| 1 | 기본 정보 | 샵 이름 + 연락처 |
| 2 | 포트폴리오 업로드 | 사진 10~12장. 좋은/나쁜 사진 가이드. 3장 이상부터 다음 활성화. 단계별 격려 문구 |
| 3 | 포트폴리오 정리 | 1장씩 카테고리 분류(심플/프렌치/자석/아트) + 대표 디자인 3~5개 선택 |
| 4 | 가격 & 시간 | 카테고리별 기본 가격 + 시간 설정 |
| 5 | 추가 비용 | 제거(자샵/타샵), 연장, 추가옵션(파츠/글리터/포인트) 각각 가격 설정 |
| 6 | 고객 안내 문구 | 기본 제공 + 수정 가능 |
| END | 완료 | "상담이 쉬워질 거예요" + [고객 화면 보기] / [현장모드 시작] |

**포트폴리오 자동 축적 흐름:**
시술 완료 → 사진 촬영 → 저장 → 시술 데이터와 자동 연결 → 포트폴리오 카드 자동 생성

**주의:**
- 사진 분류: 한 장씩 보여주고 버튼 클릭 (여러 장 한꺼번에 X → 피로)
- 가격: 처음엔 대략적 기준만. 시술 데이터 쌓이면서 보정
- 대표 디자인: 전체 업로드 ≠ 현장 첫 화면 노출. 별도 선택 필요
- "건너뛰기/나중에 정리" 옵션 제공하여 완료 압박 줄이기

## 3. 문서 간 교차 설계 포인트

### 3-1. 통합 데이터 모델 필요

5개 문서에서 공통으로 사용하는 데이터:

| 데이터 | 사용처 |
|--------|--------|
| 포트폴리오 (사진 + 카테고리 + 가격) | 온보딩에서 등록, 현장모드/미리정하기에서 선택, 공유카드에서 활용 |
| 시술 기록 (디자인 + 옵션 + 최종가 + 사진) | 현장모드에서 생성, 결제에서 확정, 공유카드에서 참조 |
| 가격 설정 (카테고리별 기본가 + 옵션별 추가가) | 온보딩에서 설정, 현장모드/미리정하기에서 계산 |
| 고객 정보 (이름 + 연락처 + 시술 이력) | 현장모드/미리정하기에서 입력, 전체에서 참조 |
| 샵 정보 (이름 + 카카오/네이버/전화) | 온보딩에서 입력, 공유카드에서 자동 적용 |

### 3-2. 공유 컴포넌트 후보

현장모드와 미리 정하기에서 겹치는 UI:
- 포트폴리오 그리드 (카테고리 필터 + 이미지 선택)
- 옵션 선택 UI (제거/길이/추가옵션)
- 가격 요약 바
- 선택 내용 확인 화면

→ 공통 컴포넌트로 추출하되, 현장모드(빠른 확정)와 미리정하기(천천히 상담)의 UX 차이는 props/variant로 분기

### 3-3. 고객정보 입력 통합 정책

| 상황 | 시점 | 필수 여부 |
|------|------|-----------|
| 현장 워크인 (신규) | 결제 직전/직후 | 선택 |
| 현장 워크인 (기존) | 자동 연결 | 입력 없음 |
| 사전 링크 (미리정하기) | 예약 정보 입력 시 | 이름/연락처 필수 |
| 공유카드 → 상담 진입 | 상담 시작 시 | 선택 |

## 5. 기술 스택 (변경 없음 + 추가)

| 항목 | 기술 |
|------|------|
| 기존 | Next.js 15 + TypeScript + Tailwind v4 + Zustand + framer-motion |
| 이미지 저장 | Supabase Storage |
| 이미지 생성 (공유카드) | HTML/CSS 템플릿 + `html2canvas` (이미지 캡처/다운로드) |
| 퍼블릭 페이지 | Next.js 동적 라우트 (인증 불요) |
| 상태 관리 (퍼블릭) | localStorage (비로그인 사용자) |
