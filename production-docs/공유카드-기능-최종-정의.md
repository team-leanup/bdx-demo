# BDX 공유카드 기능 최종 정의

---

## 1. 기능 목적

공유카드는 단순 이미지 생성 기능이 아닙니다.

> 시술 결과를 "이미지 → 링크 → 상담 → 예약"으로 이어지게 만드는 기능입니다.

즉,
- 결과를 콘텐츠로 만들고
- 그 콘텐츠가 고객 유입으로 이어지게 하는 구조입니다.

---

## 2. 공유카드 전체 구조

공유카드는 2가지로 구성됩니다.

### ① 이미지 카드 (퍼지는 용도)
- 인스타그램 / 카카오톡 등에 업로드되는 이미지
- 고객이 처음 접하는 콘텐츠
- **역할: "관심 유도"**

### ② 링크 카드 (실제 페이지)
- 이미지 클릭 시 들어오는 페이지
- 상담 및 예약으로 이어지는 핵심 화면
- **역할: "행동 유도"**

> 이 2개는 반드시 분리되어야 합니다

---

## 3. 전체 흐름 (시스템 기준)

### [STEP 1] 시술 완료
- 시술 종료
- 결제 완료

### [STEP 2] 사진 저장
- 시술 결과 사진 촬영 또는 업로드

### [STEP 3] 공유카드 생성
- "공유카드 만들기" 버튼 클릭

### [STEP 4] 카드 생성 처리
시스템이 자동으로 수행:
- 시술 데이터 불러오기
- 이미지 생성
- 링크 생성

### [STEP 5] 결과 제공
샵에게 아래 2개 제공:
- 이미지 파일
- 공유 링크

### [STEP 6] 외부 공유
샵이 인스타 등에 업로드:
- 이미지 업로드
- 링크 포함

### [STEP 7] 고객 유입
고객 행동:
- 이미지 확인
- 링크 클릭

### [STEP 8] 상담 진입
- 링크 카드 페이지 진입
- 상담 시작

### [STEP 9] 예약 연결
- 카카오톡 / 네이버 / 전화로 예약

> 이 흐름이 하나의 완성 구조입니다

---

## 4. 이미지 카드 구성 (상세)

**목적**: "예쁘다"라고 느끼게 만드는 것

### 구성 요소
- 시술 결과 이미지 (가장 중요)
- 최소 텍스트 (선택) — 예: "자석 네일", "포인트 파츠"

### 포함하면 안 되는 것
- 가격
- 버튼
- 긴 설명
- 링크

> 이유: 이미지는 클릭을 유도하는 역할만 해야 함

---

## 5. 링크 카드 구성 (핵심 화면)

### 전체 구조

**① 상단**
- 시술 이미지 (1~n장)

**② 안내 문구**
> "이 디자인, 해당 샵에서 진행 가능해요"
- 목적: 고객 불안 제거

**③ 시술 요약**
- 디자인 종류
- 주요 옵션 (파츠, 글리터 등)
- 짧고 직관적으로

**④ 상담 CTA**
> [이 디자인으로 상담하기]
- 가장 중요한 버튼

**⑤ 예약 연결 영역**
- 카카오톡
- 네이버 예약
- 전화
- 샵 설정 정보 기반으로 자동 노출

---

## 6. 상담 연결 방식

**CTA 클릭 시:**
- 상담 페이지로 이동

**자동 설정:**
- 디자인 정보 자동 입력
- 옵션 정보 자동 반영

> 고객이 다시 설명하지 않도록

---

## 7. 예약 연결 방식

상담 완료 후 "예약 방법 선택" 화면 제공

**버튼:**
- 카카오톡 예약
- 네이버 예약
- 전화

**카카오톡 클릭 시 자동 메시지 생성:**
> "안녕하세요 😊
> 이 디자인으로 상담 진행했는데
> 예약 가능할까요?"
> - 공유 링크 포함

> 핵심: 샵이 추가 설명 없이 이해 가능

---

## 8. 샵 정보 연결 구조

샵은 최초 1회만 입력:
- 카카오톡 링크
- 네이버 예약 링크
- 전화번호

이 정보는:
- 모든 공유카드에 자동 적용
- 카드 생성 시마다 입력 ❌

---

## 9. 핵심 UX 포인트 (중요)

**① 공유는 선택 기능**
- 강제하면 사용 안 함
- 자연스럽게 유도

**② 생성은 즉시 완료**
- 버튼 누르면 바로 생성
- 수정 단계 최소화

**③ 상담은 자동 연결**
- 고객 입력 최소화

**④ 예약은 연결만**
- 직접 예약 기능 없음
- 기존 채널 활용

---

## 10. 기능 핵심 정리

> 공유카드는 "시술 결과 데이터를 기반으로 이미지와 링크를 동시에 생성하고, 링크를 통해 상담 → 예약으로 이어지는 구조"

---

## 11. 개발자가 이해해야 할 핵심 3가지

**① 데이터 기반 기능**
- 시술 데이터 → 카드 자동 생성

**② 두 가지 출력**
- 이미지 (콘텐츠)
- 링크 (전환)

**③ 연결 구조**
- 링크 → 상담 → 예약

---

## 12. 최종 한 줄

> **"공유카드는 시술 결과를 고객 유입으로 전환시키는 기능입니다"**
