## BDX 샵 온보딩용 포트폴리오 등록 UX (상세)

핵심 목표는 딱 3개입니다.
1. 처음 써보는 샵도 3분 안에 시작할 수 있어야 함
2. 많이 입력하게 하면 안 됨
3. 바로 현장모드에서 쓸 수 있는 최소 포트폴리오가 만들어져야 함

결론부터 말하면 초기 구조는 이렇게 가는 게 가장 좋습니다.

사진 선택 → 스타일 분류 → 가격 입력 → 대표 디자인 설정 → 완료

중요한 건 "카테고리별로 먼저 생각해서 업로드"가 아니라
"사진부터 고르게 하고, 나중에 가볍게 분류"입니다.

### 1. 첫 진입 화면

**목적:** 샵주인이 "복잡하지 않다"고 느끼게 만드는 화면입니다.

**화면 문구:**
> 샵에서 자주 하는 대표 디자인만 먼저 등록해볼게요 😊
> 사진 10장 정도만 올리면 바로 시작할 수 있어요.

**보조 문구:**
> 처음부터 완벽하게 준비하지 않아도 괜찮아요.
> 시술 후 사진은 자동으로 계속 쌓여요.

**버튼:** 대표 디자인 등록하기

**이 화면이 중요한 이유:** 여기서 "많이 해야 하나?"라는 불안을 없애야 합니다.
그래서 "10장 정도", "바로 시작", "자동으로 쌓인다" 이 세 메시지가 꼭 들어가야 합니다.

### 2. 사진 선택 화면

**목적:** 샵주인이 가장 쉽게 할 수 있는 행동부터 하게 만드는 단계입니다.

**화면 문구:**
> 샵을 대표하는 디자인 사진을 골라주세요 😊

**보조 문구:**
> 인스타 사진이나 휴대폰에 있는 사진 그대로 올려도 괜찮아요.

**화면 구성:**
- 사진 업로드 박스
- 사진 여러 장 선택 가능
- 선택된 사진 썸네일 리스트 노출

**가이드:**
- 최소 8장
- 권장 10~12장
- 최대 20장

**버튼:** 다음으로

**이 단계에서 중요한 점:** 카테고리를 먼저 묻지 않습니다.
먼저 "이 사진들 중 어떤 걸 보여주고 싶은지"만 선택하게 해야 합니다.
샵주인은 사진을 고르는 건 쉬워하지만, 분류부터 하라고 하면 귀찮아집니다.

**추천 UX:**
- 사진을 1장도 안 올리면 버튼 비활성화
- 8장 이상 선택 시 "좋아요, 바로 시작할 수 있는 정도예요 😊" 문구 노출

### 3. 스타일 분류 화면

**목적:** 업로드한 사진을 현장모드에서 쓸 수 있게 최소한으로 분류하는 단계입니다.

**핵심 원리:** 사진 하나씩 길게 설정하게 하면 안 되고 한 장씩 빠르게 버튼만 누르게 해야 합니다.

**화면 문구:** 이 디자인은 어떤 스타일인가요?

**화면 구성:**
- 선택한 사진 1장 크게 표시
- 아래에 카테고리 버튼 4개: 심플 / 프렌치 / 자석 / 아트

**보조 기능:**
- 이전 / 다음
- 현재 진행 상태 표시 예: 3 / 10

**빠른 선택용 버튼:** 건너뛰기 / 나중에 정리하기

**버튼:** 다음 사진

**이 단계가 중요한 이유:** 자동 분석은 MVP에서 신뢰를 깨뜨릴 가능성이 큽니다.
반면 이 방식은 직관적입니다. 샵주인은 사진을 보면서 "이건 자석이지", "이건 프렌치지" 정도는 바로 판단할 수 있습니다.

**중요한 디테일:** 한 화면에 여러 사진을 한꺼번에 보여주고 분류하게 하면 피로해집니다.
한 장씩 보여주고 한 번 클릭하게 해야 속도가 납니다.

### 4. 가격 입력 화면

**목적:** 포트폴리오에서 고객이 선택할 때 가격 감을 줄 수 있게 만드는 단계입니다.

여기서 중요한 포인트는 처음부터 사진마다 가격을 다 넣게 하면 안 됩니다.

그래서 추천 구조는 "기본 가격 먼저, 개별 수정은 선택"입니다.

**화면 문구:** 가격도 간단하게 정리해볼게요 😊

**1단계 — 기본 가격대 설정**

**화면 구성:** 샵 대표 가격대 입력 예: 55,000원부터

또는 카테고리별 기본 가격 입력:
- 심플 50,000원~
- 프렌치 55,000원~
- 자석 60,000원~
- 아트 70,000원~

**추천 방식:** 카테고리별 기본 가격이 가장 좋습니다.
왜냐하면 나중에 현장모드에서 고객이 사진을 볼 때 "65,000원~" 같은 식으로 자연스럽게 보여줄 수 있기 때문입니다.

**2단계 — 개별 수정은 선택**

**문구:** 특별히 가격이 다른 디자인만 수정할 수 있어요.

**버튼:** 기본 가격으로 시작하기 / 직접 수정하기

**이 구조가 중요한 이유:** 처음부터 완벽하게 가격을 맞추게 하면 시작을 못 합니다.
처음엔 대략적인 기준만 잡고, 나중에 실제 시술 데이터가 쌓이면서 보정하는 게 맞습니다.

### 5. 대표 디자인 설정 화면

**목적:** 현장모드 첫 화면에 어떤 사진이 먼저 뜰지 정하는 단계입니다.

**화면 문구:** 먼저 보여줄 대표 디자인을 골라주세요 😊

**보조 문구:** 현장에서 고객이 가장 먼저 보게 될 사진이에요.

**화면 구성:**
- 업로드한 사진들 썸네일
- 사진마다 선택 토글
- 대표 추천 / 인기 후보 표시 가능

**선택 규칙:** 최소 4개 / 권장 8개

**추천 기준:** 심플 2장 / 프렌치 2장 / 자석 2장 / 아트 2장

**이 단계가 중요한 이유:** 처음부터 모든 사진을 똑같이 노출하면 고객이 선택하기 어렵습니다.
그래서 "초기 메인 포트폴리오"를 따로 골라야 합니다.

쉽게 말하면 전체 사진 업로드와 현장 첫 화면 노출 사진은 다를 수 있어야 합니다.

### 6. 완료 화면

**목적:** "이제 바로 쓸 수 있다"는 느낌을 줘야 합니다.

**화면 문구:**
> 포트폴리오 준비가 완료되었어요 😊

**보조 문구:**
> 이제 현장에서 고객이 디자인을 바로 고를 수 있어요.
> 시술 후 사진은 자동으로 계속 추가돼요.

**버튼:** 포트폴리오 보러가기 / 현장화면 미리보기

**여기서 꼭 넣어야 할 문구:**
> 처음부터 완벽하지 않아도 괜찮아요.
> 시술이 끝날수록 포트폴리오가 더 좋아집니다.

**이 메시지가 중요한 이유:** 샵주인이 "아직 사진이 부족한데..."라고 느끼는 불안을 없애야 하기 때문입니다.

### 7. 시술 후 포트폴리오 자동 연결 흐름

**핵심 개념:** 처음엔 샵이 수동으로 씨앗을 심고, 이후부터는 시술이 끝날 때마다 자동으로 포트폴리오가 자랍니다.

**흐름:**
1. 사전 상담 또는 현장 선택
2. → 예약/시술 데이터 생성
3. → 시술 완료
4. → 에프터 사진 촬영
5. → 저장
6. → 같은 예약 데이터와 연결
7. → 포트폴리오 카드 자동 생성

**실제 화면 흐름:**

시술 완료 화면:
- **문구:** 시술 사진을 저장해볼까요? 😊
- **버튼:** 사진 촬영하기 / 나중에 저장하기

사진 촬영 후:
- **문구:** 이번 시술 기록으로 포트폴리오에 추가할게요.

**자동으로 연결되는 데이터:**
- 선택한 디자인
- 추가 옵션
- 최종 가격
- 소요 시간
- 시술 날짜

**선택 항목:**
- 대표 포트폴리오로 노출
- 고객 기록만 저장

**이 선택지가 필요한 이유:** 모든 사진이 현장 첫 화면에 바로 뜨면 안 됩니다.
어떤 건 고객 기록으로만 남기고, 어떤 건 포트폴리오에 노출할 수 있어야 합니다.

**추천 구조:** 저장 시 기본값은 "고객 기록 + 포트폴리오 저장"
단, "대표 노출"은 별도 체크 없이 나중에 관리화면에서 설정

**이유:** 시술 직후에는 직원이 바쁩니다. 그 순간 너무 많은 선택을 하게 하면 안 됩니다.

### 8. 최종 추천 온보딩 구조

**온보딩 단계:**
1. 대표 사진 10~12장 업로드
2. 사진별 스타일 분류
3. 카테고리별 기본 가격 설정
4. 대표 디자인 8개 선택
5. 완료

**운영 단계:**
1. 고객이 포트폴리오에서 선택
2. 시술 진행
3. 시술 후 사진 촬영
4. 자동 저장
5. 포트폴리오 점점 축적

### 9. 왜 이 구조가 좋은가

**샵주인 입장:**
- 처음부터 많은 입력 안 해도 됨
- 인스타 사진 재활용 가능
- 대표 사진만 올리면 바로 시작 가능

**고객 입장:**
- 사진 보면서 고를 수 있음
- 가격 감도 바로 볼 수 있음

**BDX 입장:**
- 초기 진입장벽 낮음
- 데이터는 점점 자동으로 쌓임
- 결국 포트폴리오가 자산이 됨

### 10. 냉정한 최종 판단

대표님이 처음 말씀하신 "카테고리별 3~4개씩 선택 업로드하는 게 낫지 않나?"
이 생각 자체는 맞습니다.

그런데 사용자 경험상 더 좋은 방식은

카테고리별로 먼저 생각해서 올리게 하는 것이 아니라
