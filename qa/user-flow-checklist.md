> 작성일: 2026-03-12
> 대상: BDX (Beauty Decision eXperience) 네일샵 상담 플랫폼
> 범위: 사장님(Owner) + 고객(Customer) 전체 플로우

## 상태 범례

`⬜ 미검증` · `🟢 통과` · `🔴 실패` · `🟡 부분통과` · `⏭️ 스킵`

## 우선순위

`🔴 Critical` — 핵심 플로우 차단, 데이터 손실/불일치
`🟠 Major` — 기능 동작하나 UX 심각한 문제
`🟡 Minor` — 사소한 UI/텍스트 이슈

## A. 인증 & 온보딩 (사장님)

### TC-A01. 이메일 회원가입 → 샵 생성 → 온보딩 완료 `🔴 Critical`

| 항목 | 내용 |
|------|------|
| **시나리오** | 신규 사장님이 이메일/비밀번호로 가입 후 샵을 설정하고 홈에 도달 |
| **사전 조건** | 미등록 이메일 |
| **테스트 단계** | 1. `/signup` 접속 → 매장명/이름/이메일/비밀번호 입력 → 가입<br>2. 자동으로 `/onboarding` 이동 확인<br>3. 온보딩 7단계 순서대로 진행 (매장정보→서비스→가격→추가금→시간→테마→완료)<br>4. "홈으로 이동" 클릭 → `/home` 도달 |
| **기대 결과** | 모든 온보딩 데이터 DB 저장, `onboardingCompletedAt` 설정, 홈 정상 표시 |
| **확인 포인트** | - 이미 등록된 이메일로 가입 시도 → "이미 등록된 이메일" 에러<br>- 비밀번호 6자 미만 → 유효성 에러<br>- 온보딩 중간 이탈(브라우저 닫기) → 재로그인 시 `/onboarding`으로 복귀 |
| **상태** | ⬜ 미검증 |
